# 여름계절학기 컴퓨터네트워크 채팅프로그램

멀티스레드 기반 TCP 채팅 프로그램입니다. 서버는 여러 클라이언트의 접속을 동시에 처리하고, 클라이언트는 송신 스레드와 수신 스레드를 분리해 사용자가 메시지를 입력하는 동안에도 서버에서 오는 메시지를 실시간으로 받을 수 있습니다.

## 파일 구성

| 파일 | 설명 |
| --- | --- |
| `multi_server.cc` | 다중 클라이언트 접속, 메시지 브로드캐스트, 닉네임 관리, 귓속말 처리를 담당하는 서버 |
| `multi_client.cc` | 사용자 입력 처리, 서버 메시지 수신, 명령어 처리 기능을 가진 클라이언트 |
| `server` | 서버 실행 파일 |
| `client` | 클라이언트 실행 파일 |
| `quakcheol.txt` | 이모티콘 아스키아트 파일(이 파일을 반드시 코드와 같이 폴더에 넣어야 합니다.) |

## 빌드 방법

```bash
g++ multi_server.cc -o server -pthread
g++ multi_client.cc -o client -pthread
```

`pthread`를 사용하므로 컴파일할 때 `-pthread` 옵션이 필요합니다.

## 실행 방법

먼저 서버를 실행합니다.

```bash
./server <PORT>
```

예시:

```bash
./server 9000
```

다른 터미널에서 클라이언트를 실행합니다.

```bash
./client <SERVER_IP> <PORT>
```

예시:

```bash
./client 127.0.0.1 9000
```

클라이언트 실행 후 닉네임을 입력하면 채팅에 참여할 수 있습니다.

## 주요 기능

### 1. 다중 클라이언트 채팅

서버는 클라이언트가 접속할 때마다 별도의 스레드를 생성합니다. 한 클라이언트가 일반 메시지를 보내면 서버는 보낸 사람을 제외한 다른 클라이언트들에게 다음 형식으로 메시지를 전달합니다.

```text
닉네임: 메시지
```

### 2. 입장 및 퇴장 알림

클라이언트가 접속하면 모든 사용자에게 입장 메시지가 전달됩니다.

```text
닉네임님이 입장하셨습니다.
```

클라이언트가 `/quit` 명령을 입력하거나 연결이 종료되면 다른 사용자들에게 퇴장 메시지가 전달됩니다.

```text
닉네임님이 퇴장하셨습니다.
```

### 3. 닉네임 변경

클라이언트에서 다음 명령어로 닉네임을 변경할 수 있습니다.

```text
/rename <새닉네임>
```

서버는 닉네임 목록을 갱신한 뒤 모든 클라이언트에게 변경 사실을 알립니다.

```text
기존닉네임님이 새닉네임(으)로 닉네임을 변경했습니다.
```

### 4. 귓속말

특정 사용자에게만 메시지를 보내는 기능입니다.

```text
/whisper <닉네임>
```

명령어를 입력하면 클라이언트에 다음 프롬프트가 표시됩니다.

```text
<닉네임>에게 할 귓속말을 입력하세요:
```

입력한 메시지는 지정한 사용자에게만 전달됩니다. 받은 사용자는 귓속말임을 알 수 있도록 다음 형식으로 출력됩니다.

```text
[귓속말] 보낸사람님: 메시지
```

보낸 사람에게는 전송 확인 메시지가 출력됩니다.

```text
[귓속말 전송] 받는사람님에게: 메시지
```

존재하지 않는 닉네임을 입력하면 다음 메시지가 출력됩니다.

```text
>> 닉네임님을 찾을 수 없습니다.
```

### 5. 이모지 전송

`/emoji` 명령어를 사용해 정해진 이름의 이모지를 채팅 메시지로 보낼 수 있습니다.

```text
/emoji <이름>
```

지원하는 이모지 이름:

| 명령어 | 출력 |
| --- | --- |
| `/emoji happy` | 😊 |
| `/emoji sad` | 😢 |
| `/emoji angry` | 😡 |
| `/emoji love` | 😍 |
| `/emoji laugh` | 😂 |
| `/emoji cry` | 😭 |
| `/emoji surprise` | 😮 |
| `/emoji thumbs` | 👍 |

알 수 없는 이모지 이름을 입력하면 사용 가능한 목록을 안내합니다.

### 6. 글자색 변경

클라이언트 화면에 출력되는 글자색을 변경할 수 있습니다.

| 명령어 | 설명 |
| --- | --- |
| `/color r` | 빨간색으로 변경 |
| `/color g` | 초록색으로 변경 |
| `/color b` | 파란색으로 변경 |
| `/color -` | 기본색으로 변경 |

### 7. 도움말

사용 가능한 명령어 목록을 확인할 수 있습니다.

```text
/help
```
### 8. 이모티콘(아스키아트)

`quakcheol.txt`에 저장된 아스키아트 이모티콘을 채팅방에 전송할 수 있습니다. 서버가 파일에서 이모티콘 번호를 찾아 해당 아스키아트를 읽고, 출력이 깨지지 않도록 한 줄을 비운 뒤 전체 클라이언트에게 전달합니다.

| 명령어 | 설명 |
| --- | --- |
| `/emoticon <번호>` | 해당 번호의 아스키아트 이모티콘 출력 |
| `/emoticonlist` | 사용 가능한 이모티콘 번호와 이름 출력 |

사용 가능한 이모티콘 목록:

| 번호 | 이름 |
| --- | --- |
| `1` | `음 망했군....` |
| `2` | `불가능` |
| `3` | `꺼억` |
| `4` | `따봉` |
| `5` | `삽가능` |
| `6` | `몽둥이` |

예시:

```text
/emoticon 3
```

출력 형식:

```text
Alice님이 3번 이모티콘 '꺼억'을 보냈습니다.

<아스키아트>
```

목록 확인 예시:

```text
/emoticonlist
```

```text
>> 사용 가능한 이모티콘 목록
1. 음 망했군....
2. 불가능
3. 꺼억
4. 따봉
5. 삽가능
6. 몽둥이
```

### 9. 접속자 목록 확인

현재 채팅방에 누가 접속해 있는지 확인할 수 있습니다.

```text
/who
```

출력 예시:

```text
>> 현재 채팅방 인원: 3/64
>> 접속자 목록
   - Alice
   - Bob
   - KSH
```

## 구현 구조

### 서버

서버는 접속한 클라이언트의 소켓과 닉네임을 배열로 관리합니다.

- `clnt_socks`: 클라이언트 소켓 목록
- `clnt_nickname`: 클라이언트 닉네임 목록
- `clnt_id`: 현재 접속한 클라이언트 수
- `pthread_mutex_t mutx`: 여러 스레드가 클라이언트 목록에 동시에 접근하지 않도록 보호

주요 처리 흐름:

1. `accept()`로 클라이언트 접속 수락
2. 클라이언트가 처음 보낸 닉네임 저장
3. 새 스레드에서 해당 클라이언트의 메시지 처리
4. 일반 메시지는 다른 클라이언트들에게 브로드캐스트
5. `/rename`, `/whisper`, `/emoticon`, `/emoticonlist`, `/who`, `/quit`은 서버에서 별도 명령으로 처리

### 클라이언트

클라이언트는 두 개의 스레드로 동작합니다.

- 송신 스레드: 사용자 입력을 읽고 서버로 전송
- 수신 스레드: 서버에서 오는 메시지를 계속 읽어 화면에 출력

`process_command()` 함수에서 `/color`, `/rename`, `/whisper`, `/emoji`, `/emoticon`, `/emoticonlist`, `/who`, `/quit`, `/help` 명령어를 처리합니다.

## 실행 예시

```text
닉네임을 입력하세요: Alice
>> 서버 연결 성공! 채팅을 시작하세요. (종료하려면 q 입력)
KSH님이 입장하셨습니다.
나: 안녕하세요
KSH: 반갑습니다
```

귓속말 예시:

```text
/whisper KSH
KSH에게 할 귓속말을 입력하세요: 이 메시지는 KSH에게만 보여요
[귓속말 전송] KSH님에게: 이 메시지는 KSH에게만 보여요
```

KSH 화면:

```text
[귓속말] Alice님: 이 메시지는 KSH에게만 보여요
```
